Quicksort compare analysis

Posted on 2017-03-10 | Edited on 2018-12-16 | In algorithm

Quicksort user  2NlnN~2NlnN 2NlnN compares to sort
an array of N distinct keys on average
(one-sixth that many exchanges)

Proof

Let CNC_NC​N​​ be the average number of compares need to sort N items
with distinct values.
We have C0=C1=0C_0=C_1=0C​0​​=C​1​​=0 for N>1N>1N>1

Express the average compare

CN=N+1+(C0+C1+...+CN−2+CN−1)/N+(CN−1+CN−2+...+C0)/NC_N = N + 1 + (C_0+C_1+...+C_{N-2}+C_{N-1})/N + (C_{N-1}+C_{N-2}+...+C_0)/NC​N​​=N+1+(C​0​​+C​1​​+...+C​N−2​​+C​N−1​​)/N+(C​N−1​​+C​N−2​​+...+C​0​​)/N
  1. cost of partitioning
    N-1 compares two partition with flag
    1 more compare in each part breakpoint compare

  1. left subarray sort

    (C0+C1+...+CN−2+CN−1)/N(C_0+C_1+...+C_{N-2}+C_{N-1})/N(C​0​​+C​1​​+...+C​N−2​​+C​N−1​​)/N

  2. right subarray sort

    (CN−1+CN−2+...+C1+C0)/N(C_{N-1}+C_{N-2}+...+C_1+C_0)/N(C​N−1​​+C​N−2​​+...+C​1​​+C​0​​)/N

Rearrange terms
